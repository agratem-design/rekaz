

## خطة تحسين حركات الخزينة

### المشكلة الحالية
- الحذف مقيد فقط بالحركات اليدوية (`reference_type === "manual"`)
- لا يمكن تعديل أي حركة
- إضافة الحركات محدودة بالإيداع فقط - لا يمكن إضافة سحب أو أنواع أخرى مباشرة

### التعديلات المطلوبة

#### 1. السماح بحذف جميع الحركات
- إزالة القيد `isManual` من `deleteTransactionMutation` الذي يمنع حذف الحركات غير اليدوية
- إظهار زر الحذف لجميع الحركات بدلاً من عرض "—" للحركات المرتبطة
- عند حذف حركة مرتبطة بمشتريات أو تسديدات، يتم تحديث الرصيد تلقائياً عبر trigger `auto_sync_treasury_balance`

#### 2. إضافة إمكانية تعديل الحركات
- إضافة زر تعديل (قلم) بجانب زر الحذف لكل حركة
- عند الضغط، يُفتح Dialog يحتوي على: المبلغ، النوع (إيداع/سحب)، المصدر، تفاصيل المصدر، الوصف، التاريخ، الملاحظات، العمولة
- تنفيذ `updateTransactionMutation` يقوم بتحديث السجل في `treasury_transactions` - الرصيد يتحدث تلقائياً عبر trigger

#### 3. توسيع نموذج إضافة حركة جديدة
- تحويل Dialog الإيداع الحالي إلى Dialog شامل "إضافة حركة"
- إضافة اختيار النوع: إيداع أو سحب
- إضافة قائمة المصادر المتاحة في النظام: مشتريات، مصروفات، تسديد زبون، تسديد مورد، مستحقات فني، إيجار معدات، نقل، راتب، استرداد، يدوي
- إضافة حقل العمولة
- الحفاظ على الحقول الحالية: المبلغ، المصدر، التفاصيل، التاريخ، الملاحظات

### الملفات المتأثرة
- `src/pages/TreasuryDetail.tsx` — التعديل الرئيسي (حذف/تعديل/إضافة)

### التفاصيل التقنية
- `deleteTransactionMutation`: إزالة شرط `isManual` check، السماح بحذف أي حركة
- إضافة state جديد `editingTx` و`editForm` للتعديل
- إضافة `updateTransactionMutation` يستخدم `supabase.from("treasury_transactions").update()`
- تعديل `depositForm` ليشمل حقل `type` (deposit/withdrawal) وحقل `commission`
- تحديث `depositMutation` ليدعم الإيداع والسحب
- الـ trigger `auto_sync_treasury_balance` الموجود سيتكفل بتحديث أرصدة الخزائن تلقائياً عند أي insert/update/delete
