# 📋 توثيق منظومة إدارة المشاريع والمقاولات

## 📌 نظرة عامة

منظومة متكاملة لإدارة المشاريع والمقاولات مبنية بتقنيات حديثة (React + TypeScript + Supabase) مع دعم قاعدة بيانات MySQL خارجية. تغطي المنظومة جميع جوانب إدارة المشاريع من التخطيط والتنفيذ إلى المالية والتقارير.

---

## 🌟 مميزات المنظومة

### 1. إدارة المشاريع
- إنشاء وإدارة مشاريع متعددة مع تتبع الحالة والتقدم
- تقسيم المشاريع إلى مراحل (Phases) مع ترقيم تلقائي
- ربط المشاريع بالعملاء والمهندسين والموردين
- تتبع الميزانية والمصروفات الفعلية
- صور المشاريع والملاحظات

### 2. إدارة بنود المشروع
- إضافة بنود متعددة لكل مشروع
- دعم أنواع قياس متعددة (طولي، مسطح، حجمي، مخصص)
- معادلات حساب مخصصة (Formula)
- ربط البنود بالمراحل والمهندسين
- تتبع نسبة الإنجاز لكل بند

### 3. الإدارة المالية
- **الخزائن**: إدارة خزائن متعددة (نقدي، بنكي) مع تتبع الأرصدة
- **المشتريات**: تسجيل فواتير المشتريات مع ربطها بالموردين والمشاريع
- **المصروفات**: تتبع جميع أنواع المصروفات
- **الإيرادات**: تسجيل الدخل من مختلف المصادر
- **مدفوعات العملاء**: نظام مدفوعات متقدم مع توزيع الدفعات على المراحل
- **التحويلات**: إدارة السلف والتحويلات المالية
- **التدفق النقدي**: توقعات التدفق النقدي الشهرية

### 4. العقود
- إنشاء عقود مخصصة مع قوالب بنود
- ربط العقود بالمشاريع والعملاء
- طباعة العقود بتصميم احترافي
- إدارة بنود العقد وشروطه

### 5. الموارد البشرية
- **المهندسون**: إدارة بيانات المهندسين (مقيم/استشاري)
- **الفنيون**: إدارة الفنيين مع أسعار متعددة (ساعة/يوم/متر/قطعة)
- **الموظفون**: إدارة بيانات الموظفين والرواتب
- **تتبع إنجاز الفنيين**: سجل الإنجاز اليومي لكل فني

### 6. العملاء والموردون
- إدارة قاعدة بيانات العملاء والموردين
- تتبع المشتريات والمدفوعات لكل مورد
- ربط الموردين بالمشاريع
- سجل أنشطة العملاء

### 7. المعدات والمخازن
- **المعدات**: إدارة المعدات مع تتبع الحالة والكميات
- **تأجير المعدات**: نظام تأجير مع حساب التكلفة التلقائي
- **المخازن**: إدارة المواد مع حركات الدخول والخروج
- **تنبيهات المخزون**: إشعارات عند انخفاض المخزون

### 8. العهد (Custody)
- إدارة العهد المالية للمهندسين والموظفين
- تتبع المصروف والمتبقي من كل عهدة
- ربط العهد بالمشاريع

### 9. ضبط الجودة
- قوائم فحص الجودة مع تقييم شامل
- ربط الفحوصات بالمشاريع والمراحل
- تتبع حالة كل بند فحص

### 10. إدارة المخاطر
- سجل المخاطر مع تقييم الاحتمالية والأثر
- خطط التخفيف والطوارئ
- متابعة حالة المخاطر

### 11. أوامر التغيير (Variation Orders)
- تسجيل وتتبع أوامر التغيير
- ربطها بالمشاريع والمراحل
- تتبع المبالغ الأصلية والمعتمدة

### 12. الجدولة الزمنية
- جدول زمني لكل مشروع مع مهام متعددة
- تواريخ مخطط وفعلي وأساسي (Baseline)
- تتبع نسبة الإنجاز والتكلفة

### 13. التقارير والطباعة
- تقارير مالية شاملة
- تقارير المشاريع التفصيلية
- تصميم طباعة مخصص (ألوان، خطوط، شعار)
- دعم خلفيات مخصصة للعقود والتقارير

### 14. إدارة المستخدمين والصلاحيات
- نظام أدوار متعدد: مدير، مهندس، محاسب، مشرف
- حماية الصفحات حسب الدور
- سجل المراجعة (Audit Log) لجميع العمليات

### 15. إدارة قاعدة بيانات MySQL
- ربط بقاعدة MySQL خارجية أو محلية
- إنشاء هيكل القاعدة تلقائياً
- حذف البيانات أو الجداول مع تأكيد مزدوج
- دعم الوضع المحلي (Windows) والسحابي

## 🔌 تعليمات ربط قاعدة بيانات MySQL

### الخطوة 1: تثبيت MySQL على Windows

1. حمّل MySQL Installer من الموقع الرسمي:
   ```
   https://dev.mysql.com/downloads/installer/
   ```

2. شغّل المثبت واختر **MySQL Server** و **MySQL Workbench**

3. أثناء التثبيت:
   - اختر **Server Configuration Type**: Development Computer
   - أنشئ كلمة مرور لحساب `root`
   - فعّل الخيار: **Configure MySQL Server as a Windows Service**
   - فعّل: **Start the MySQL Server at System Startup**

4. بعد التثبيت، تأكد أن MySQL يعمل:
   ```cmd
   mysql -u root -p
   ```

### الخطوة 2: إنشاء قاعدة بيانات جديدة

افتح MySQL Workbench أو Command Line وأنشئ قاعدة بيانات:

```sql
CREATE DATABASE project_management 
  CHARACTER SET utf8mb4 
  COLLATE utf8mb4_unicode_ci;
```

> ⚠️ يجب استخدام `utf8mb4` لدعم اللغة العربية بشكل كامل.

### الخطوة 3: تشغيل السيرفر المحلي

1. افتح Command Prompt أو PowerShell

2. انتقل لمجلد السيرفر المحلي:
   ```cmd
   cd mysql-local-server
   ```

3. ثبّت التبعيات:
   ```cmd
   npm install
   ```

4. شغّل السيرفر:
   ```cmd
   npm start
   ```
   أو:
   ```cmd
   node index.js
   ```

5. يجب أن يظهر:
   ```
   🚀 MySQL Local Server is running on http://localhost:3456
   📡 API Endpoint: POST http://localhost:3456/api/mysql-manager
   ```

> 💡 اترك نافذة الـ Command Prompt مفتوحة أثناء الاستخدام.

### الخطوة 4: الربط من المنظومة

1. افتح المنظومة في المتصفح
2. اذهب إلى **إدارة قاعدة بيانات MySQL** من القائمة الجانبية
3. فعّل **الوضع المحلي** من المفتاح في أعلى الصفحة
4. أدخل بيانات الاتصال:
   - **عنوان السيرفر**: `localhost`
   - **المنفذ**: `3306`
   - **اسم المستخدم**: `root`
   - **كلمة المرور**: (كلمة المرور التي أنشأتها)
   - **اسم قاعدة البيانات**: `project_management`
5. اضغط **اختبار الاتصال**
6. إذا نجح، اضغط **إنشاء الهيكل** لإنشاء جميع الجداول

### الخطوة 5: تصدير ملف SQL (اختياري)

يمكنك تصدير ملف SQL يحتوي على هيكل قاعدة البيانات الكامل:
1. اضغط زر **تصدير SQL** في صفحة إدارة MySQL
2. سيتم تحميل ملف `.sql` يمكنك تشغيله يدوياً في MySQL

## 🔧 استكشاف الأخطاء

### مشكلة: فشل الاتصال بـ MySQL

**الحل:**
1. تأكد أن MySQL Server يعمل:
   ```cmd
   sc query MySQL80
   ```
2. تأكد من صحة بيانات الاتصال (اسم المستخدم وكلمة المرور)
3. تأكد أن المنفذ 3306 غير محجوب من جدار الحماية
4. جرّب الاتصال من MySQL Workbench أولاً

### مشكلة: السيرفر المحلي لا يعمل

**الحل:**
1. تأكد أن Node.js مثبت: `node --version`
2. تأكد من تثبيت التبعيات: `cd mysql-local-server && npm install`
3. تأكد أن المنفذ 3456 غير مستخدم
4. جرّب منفذ آخر بتعديل `PORT` في `index.js`

### مشكلة: خطأ في إنشاء الهيكل

**الحل:**
1. تأكد أن قاعدة البيانات موجودة ومنشأة بـ `utf8mb4`
2. تأكد أن المستخدم لديه صلاحيات `CREATE TABLE`
3. راجع سجل العمليات في الصفحة للتفاصيل

### مشكلة: أحرف عربية تظهر بشكل غير صحيح

**الحل:**
1. تأكد أن قاعدة البيانات منشأة بـ:
   ```sql
   CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci
   ```
2. تأكد من إعدادات MySQL:
   ```ini
   [mysqld]
   character-set-server=utf8mb4
   collation-server=utf8mb4_unicode_ci
   ```

### مشكلة: الوضع السحابي لا يتصل بـ localhost

**الحل:**
هذا طبيعي! Edge Functions تعمل على خوادم Supabase ولا تستطيع الوصول لجهازك. استخدم **الوضع المحلي** بدلاً من ذلك.
